Process Groups

The PMP exam focuses on 5 Process Groups, which are logical groupings of project management processes:

  1. Initiating Process Group

    Defines and authorizes the project or phase.

    Key Processes:

    • Develop Project Charter
    • Identify Stakeholders

    Key Inputs:

    • Business case
    • Agreements

    Tools and Techniques:

    • Expert judgment
    • Data gathering techniques

    Key Outputs:

    • Project charter
    • Stakeholder register

  2. Planning Process Group

    Establishes the scope, objectives, and course of action.

    Key Processes:

    • Develop Project Management Plan
    • Plan Scope Management
    • Collect Requirements
    • Define Scope
    • Create WBS

    Key Inputs:

    • Project charter
    • Enterprise environmental factors

    Tools and Techniques:

    • Facilitated workshops
    • Decomposition

    Key Outputs:

    • Project management plan
    • Project scope statement

Process Group Interactions

Process Groups are not linear and often overlap. For example:

  • Planning processes may need to be revisited as the project progresses.
  • Monitoring and Controlling processes occur throughout the project lifecycle.
  • Closing processes may trigger the Initiating processes for a new phase.

Practice Question

Which Process Group is responsible for developing the project management plan?

  1. Initiating
  2. Planning
  3. Executing
  4. Monitoring and Controlling

Answer: b. Planning

Explanation: The Planning Process Group is responsible for developing the project management plan, which outlines how the project will be executed, monitored, and controlled.
